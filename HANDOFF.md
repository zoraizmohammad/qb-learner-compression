## 1. Goal — what we are trying to build

A small **hybrid quantum–classical "Bayesian learner"** that does binary categorization on
a Pothos–Chater–style 2-feature toy dataset, and a study of **how far its circuit can be
compressed under NISQ hardware constraints** before accuracy collapses.

The intended scientific contributions (as written in the paper):

1. A density-matrix belief state on 2 qubits, updated per stimulus by a **Kraus/CP evidence
   channel** (a "quantum Bayesian update").
2. A **Heisenberg-type variational ansatz** (RXX·RYY·RZZ entanglers) on a connectivity
   compatible with IBM's `FakeManila`.
3. A **hardware-aware loss** `L = L_CE + λ·N₂q`, where `N₂q` is the number of two-qubit
   gates **after transpiling to FakeManila** (so SWAP overhead from the coupling map counts).
4. A **learnable binary mask** over entanglers, giving an empirical
   accuracy-vs-CNOT Pareto frontier, plus a **learned-vs-random mask ablation** showing that
   *structured* sparsity beats unstructured sparsity at equal gate budget.

Course context: ECE final project, Duke. Repo: https://github.com/zoraizmohammad/qb-learner-compression

**Net:** the simulator/Kraus/density-matrix plumbing is genuine and runs, but (a) the optimizer
is not learning (chance accuracy), (b) pruning never triggers, (c) the three pillars the paper
sells (FakeManila penalty, Heisenberg ansatz, learned-mask Bayesian learner) are absent / not the
default / not as described, and (d) **Table I, the λ-sweep table, and the ablation table appear
hand-entered — they do not trace to any logged file.** Three different "authoritative" summary
CSVs disagree with each other on N₂q and CE for the same experiment names.

### Why the model is stuck at chance (root-cause hypotheses, highest first)
1. **Frozen, hand-weighted readout** decouples the decision from the trained circuit; the only
   trainable params are `theta`, but the logit weights `0.6/0.4/0.3/0.2` are constants. Gradient
   signal into `theta` is weak/uninformative.
2. **Per-sample re-initialized belief** + weak post-unitary channel → the "evidence" barely moves
   the state; there is no accumulation, so nothing distinguishes class A from B except the
   feature-encoding rotations feeding a fixed head.
3. **Finite-difference gradients** (`h=1e-5`, one forward pass per param) on a nearly-flat loss →
   tiny, noisy updates; effectively no learning over 100 steps.
4. **Penalty never changes the mask**: `lambda` is added to the scalar loss but the optimizer only
   touches `theta`; mask changes only via `greedy_prune`, whose tolerance evidently never accepts a
   removal (or never runs in the sweep), so sparsity stays 1.0.

## 5. Failed attempts / dead ends (so they aren't repeated)

- **Maximally-mixed prior `I/2ⁿ` was tried and abandoned** — it's invariant under the unitary, so
  the ansatz had zero effect on it; replaced by the hardcoded `0.55|00⟩ + 0.45 I/d` bias to force a
  gradient signal (see `learner.py:597-602` docstring). This is a patch over the real problem (the
  readout/structure, not the prior).
- **Amplitude-damping evidence channel** (the documented default in `channels.py`) was silently
  overridden to `projective` in `learner.py:253` because it "creates better separation" — i.e., the
  channel was being tuned for classification, not derived from the Bayesian story.
- **The λ penalty alone does not induce compression** — adding `λ·N₂q` to a scalar loss optimized
  only over `theta` cannot change the discrete mask; the sweep data shows sparsity pinned at 1.0.
- **Many "final_large" batches were re-run** (multiple timestamped dirs with the same 13-experiment
  grid) and all produced byte-identical chance-level metrics, confirming config variations
  (depth4, channel08, expressive) had no effect because training never converged.
- Git commit `52fcca1 "more checks for too normalized results need to investigate why"` is the
  earlier acknowledgement of this same symptom.
